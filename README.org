* My Writing System

I write in Emacs [[https://orgmode.org][org-mode]] and render documents to PDF for readers. This folder contains my
writing system, built on [[https://pandoc.org][pandoc]] and [[https://typst.app][typst]]. I use macOS but this should work on Linux with
minor tweaks.

Run =./setup.sh= to install all dependencies (pandoc, typst, uv, IBM Plex fonts). Then
write. When you are done, run =make= with your filename (or use =M-x compile= in Emacs) to produce
a PDF. Put visuals in =images/= and reference them in your Org content.

#+name: fig-pipeline
#+caption: The rendering pipeline from Org source to PDF output.
[[./images/pipeline.svg]]

The Makefile finds the matching =.org= file, runs pandoc with the Lua filter (=ajd.lua=) and Typst
template (=style/ajd.typ=), writes intermediate Typst to =.cache/=, and compiles the PDF into
=docs/=. It declares both as dependencies, so changes to either trigger a rebuild. You can
adjust typography and rendering choices by editing the template in =style/=.

The author defaults to yours truly and the date defaults to the build date. Override either in the
Org file header with =#+author:= or =#+date:=. The title appears in a running header on every page
alongside the page number. The Lua filter rewrites image paths from relative to absolute so Typst
can locate them regardless of build directory.

** Formatting

Text can be *bold*, /italic/, _underlined_, +struck through+, or =inline code=. Links render in
color, like [[https://www.poetryfoundation.org/poems/46271/i-am-a-parcel-of-vain-strivings-tied][this one]]. Block quotes look like the following:

#+begin_quote
I am a parcel of vain strivings tied by a chance bond together.

-- Henry David Thoreau
#+end_quote

Tables follow the booktabs convention with horizontal rules and no vertical lines:

*** More features

Third level headings like this one are run-in headings. They flow into the paragraph text and are
useful for minor divisions without adding vertical space. The template also supports two-column
layouts, aside blocks, and code with syntax highlighting.

#+begin_twocol
Two-column layouts use =#+begin_twocol= blocks. A horizontal rule (=-----=) marks the column break.
This side can contain narrative text or context that pairs with the content on the right.
-----

#+begin_aside
Asides provide tangential context. They are visually quieter than callouts and work well for
background information that does not belong in the main text.
#+end_aside

#+end_twocol

#+begin_src python
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b
#+end_src

Footnotes render at the bottom of the page with a short separator rule [fn:1].
