My Writing System

Aldrin D'Souza

2026-03-29

I write in Emacs org-mode and publish documents as both PDF and HTML. This repository contains my writing system, built on pandoc and typst. I use macOS but this should work on Linux with minor tweaks.

Run ./setup.sh to install all dependencies (pandoc, typst, uv, IBM Plex fonts). Then write. When you are done, run make with your filename (or use M-x compile in Emacs) to produce both outputs. Put visuals in images/ and reference them in your Org content.

The Makefile drives two parallel pipelines from each .org file. For PDF, pandoc converts the Org source to Typst markup via the Lua filter (ajd.lua) and template (style/ajd.typ), writes intermediate Typst to .cache/, and compiles the PDF into docs/. For HTML, pandoc converts the same source directly to a standalone HTML page styled by style/document.css. SVG images are inlined into the HTML so each page is self-contained. Run make site to build every .org file in the directory.

The author defaults to yours truly and the date defaults to the build date. Override either in the Org file header with #+author: or #+date:. Setting either to blank suppresses it in the HTML output. The Lua filter rewrites image paths per format: root-relative for Typst, inlined SVGs for HTML.

Formatting

Text can be bold, italic, underlined, struck through, or inline code. Links render in color, like this one. Block quotes look like the following:

I am a parcel of vain strivings tied by a chance bond together.

– Henry David Thoreau

Tables follow the booktabs convention with horizontal rules and no vertical lines:

The plots/ visualization scripts render the same data in the grayscale broadsheet style.

Cross-reference a table or figure by adding #+name: label above it and writing [[#label]] in your text. Numbers update automatically: #tbl-taxi, #fig-taxi-trips, #fig-taxi-heatmap.

More features

Third level headings like this one are run-in headings. They flow into the paragraph text and are useful for minor divisions without adding vertical space. The template also supports two-column layouts, aside blocks, and code with syntax highlighting.

Two-column layouts use #+begin_twocol blocks. A horizontal rule (-----) marks the column break. This side can contain narrative text or context that pairs with the content on the right.

Asides provide tangential context. They are visually quieter than callouts and work well for background information that does not belong in the main text.

def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b

Footnotes render at the bottom of the page with a short separator rule 1.
